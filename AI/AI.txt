-> Context Window
    - functions like short term memory will help us to pass our tech corp (local) data documents to context window
    - upper bound of tokens for xai and anthropic is around 200k but for gemini it is 1M
    - nano mini and flash model can hold 2-4k tokens around 1.5-3k words 
    - as we are not able to provide GBs of to context window becuuse of its max token limitation , Embedding comes into picture
-> Embedding
    - embedding is a technique used to convert high-dimensional data (like text or images) into a lower-dimensional representation (like a vector or matrix).
    - In this we use vector DB instead of sql DB

-> LangGraph
    - this extends langchain to handle more complex multistep workflows that can go beyond simple question and answer interactions
-> RAG 
    - Retrival Augmented Generation
        - Retrieval Augmented Generation is an architecture where an LLM does not rely only on its training data. Instead, it retrieves relevant external documents at runtime and provides them to the model as context before generating an answer.
        - Augumentaion refers to the process, where the retrieve data ( our corporate docs stored as vecotor db) is injected into the prompt at runtime
    - RAG is a very powerful system that can instantly improve the depth of the knowledge beyond its training data

-> Git worktrees allow multiple working directories from the same repository. Each directory can check out a different branch. This is useful for isolated agents, parallel development, or running multiple tools (like AI agents) without branch conflicts.
    - All share the same .git history, but have separate working directories.
    - Since all worktrees share the same Git repository, changes are visible immediately after commits.
    - git worktree add .worktrees/agent1 -b agent1
    - git worktree add .worktrees/agent2 -b agent2
    - git worktree add .worktrees/agent3 -b agent3
