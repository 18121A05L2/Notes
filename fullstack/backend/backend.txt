-> Express 
    - Route → Middleware → Controller → Service → Repository → Database
    - Repositories are used to separate data access logic from business logic, making the codebase cleaner, testable, and easier to scale or change databases in the future
    - Controllers :: Functions that handle actual request logic
    - Services :: Business logic that should NOT be tied to Express
    - Jobs :: Background tasks like cronjobs
-> MongoDb :: A NoSQL, document-oriented database storing data in BSON (binary JSON) format.
-> Swagger is an opensource API documentation framework
    - swagger is a set of rules or a specification for a format describing REST APIs
-> Running Express and Socket.IO on the same domain and port ensures that cookies are automatically shared with Socket.IO connections.
-> app.use(bodyParser.urlencoded({ limit: "50mb", extended: true }));
    - This parses data from HTML forms (URL-encoded format).
-> A middleware is a function that runs before the main controller of a route.
    - Middleware performs essential filters like: Authentication , Authorization (your case) , Validation ,Rate limiting , Logging , Sanitization
    - we can stack any number of middle wares
    - Difference between Global Middleware and Route Middleware
-> Winston is the most popular logging library for Node.js.
-> Redis cache is an opensource tool that runs as a service in the background that allows you to store data in memory for high performance data retrival and storage purposes
    - cache hit will get data from redis whereas the cache miss will get the data from db and then store it in redis
    - cache worker :: This will monitor the database for any data change and if something changes then cache worker will populate that data in redic cache ( so with this we dont have any stale data in cache )
    - Remote Dictionary Server
-> Multer is Express middleware used to handle: multipart/form-data , file uploads from forms
    - multer.memoryStorage() -  This setup allows your backend to receive a single or multiple uploaded files, keep it temporarily in memory, and then process or upload it to cloud storage.
-> SMTP is only to send Emails and IMAP or POP3 is to receive Emails
    - This uses TCP to send data
-> MongoDB string matching is case-sensitive by default.   
-> PM2 is a production process manager for Node.js applications.
    - It is used to run, monitor, restart, and manage backend services reliably in production.
-> Idempotency 
    - A request method is considered idempotent if the intended effect on the multiple identical requests with that method is the same as effect for a single such request.
    - POST and PATCH ( depending on the implementation ) are not idempotent
        - with Idempotency-Key in the requrest header, Same request is not processed twice
-> PATCH modifies only the fields provided in the request body, instead of replacing the entire resource.
